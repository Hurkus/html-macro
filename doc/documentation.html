1. Introduction

HTML-Macro is a transpiler for HTML documents. It's main focus is to aid in the development of static websites. The main benefits of HTML-Macro are:

  1. Splitting large documents into smaller, more managable files.
  2. Generating table or grid entries based on templates.
  3. Easier generation of language variants of the document.


The program is implemented in C++, can be compiled with GCC (g++) and is used as a command-line tool. It synergises well with make.




2. Command-line interface

The program requires a path to one input file, which will be parsed, processed and then piped via stdout.


General usage would be: html-macro [options] [variables] <path>, where:

  • [options] represents the optional list of program options described below.
  • [variables] represents the list of predefined variables used in expressions. They are written in the form of [name]=[value].
  • <path> represents the mandatory path to the input file (typically a .html file). The path cannot contain an = character, as that would be considered a variable assignment.


List of program [options]:

Long option Short option Description
--help -h Print program information and a short manual/list of all options.
--include <path> -i <path> Add path to the list of paths that are searched when including files using a relative path with the <INCLUDE> element macro. The path must lead to an existing directory.
--output <path> -o <path> Write output directly to a file instead of stdout. Errors, warnings and general information are still piped to stderr.
-x Discard the stdout stream and/or do not write any output to a file. This option can be used to check for errors without generating any output.



3. Parsing

There are 4 different file types considered during parsing:

  • HTML files with extension .html,
  • CSS files with extension .css,
  • JavaScript files with extension .js and
  • text files with any other extension.


Currently only HTML files are actually parsed, the others are considered as plain text. This will change in a future version when CSS and JS compression is added.



3.1. HTML files

When parsing HTML files, whitespace surrounding all HTML elements is generally modified in order for the final document to aproximate a canonical HTML structure. Each element retains information about the existance of surrounding whitespace only to ensure that the resulting HTML doesn't lose/gain any unwanted whitespace. The actual text nodes, containing only whitespace, are discarded during parsing.


Example: Whitespace retention.

input output
<p>Here<u> is some </u><span>text</span>.</p>

<p>
    Long text,
    long text
</p>

<div>
<p>Weird input
</p>
</div> 		<p>Here<u> is some </u><span>text</span>.</p>
<p>
    Long text,
    long text
</p>
<div>
    <p>Weird input
    </p>
</div>

Here we can see that only existance of whitespace is preserved. This is okay because HTML is rendered in the same manner while ensuring readability of the final document.


Aditionally text within multiline text nodes is properly indented to match the desired indentation level of the parent node. This is achieved by examining the tab count of the first line after the first newline character \n and then adjusting all following lines of text accordingly. Only the tab character \t is regarded as the indentation character and not space. It is the users responisbility to provide the proper level of indentation of multiline text.


Note:

The sample code in all examples uses spaces for indentation because it is more reliable for displaying raw text. Just imagine they are actually tabs.


Example: Multiline text indentation

input output
<head>
<style>
    body {
        color: red;
    }
</style>
</head> 		<head>
    <style>
        body {
            color: red;
        }
    </style>
</head>

Here we can see how the indentation level of the text within the <style> element is calculated and adjusted based on the indentation level of the first line of text (colored with green) after the first newline character.


While most of the HTML elements and their attributes are coppied to the final document, some of them perform special operations that programatically generate content. These elements and attributes are typically written in uppercase. Details about this mechanism can be found in the chapter on macros.


Another consideration when writing regular HTML is with element attributes. The types of quotes around the attriute's value hold different meanings. Double quotes attr="..." represent a string, while single quotes attr='...' represent an expressions.


Text is also parsed for expressions within curly brackets {}, which are replaced by the stringified value of the evaluated expression. Backslashes \{...} escape the expression and output the text as is. More on expressions and string interpolation can be found in the chapter on expressions.


Special parsing cases apply for certain elements:

  • <style> – All content is parsed as plain text untill the closing </style> tag.
  • <script> – All content is parsed as plain text untill the closing </script> tag.
  • <pre> – All content is parsed as plain text untill the closing </pre> tag.




4. Macros

While most content from source files is coppied to the final document, macros can invoke special functionality for generating or excluding content. The user can also define custom macros, although with limited functionality.


There are two sets of predefined macros:

  • Element macros — triggered by the element's tag.
  • Attribute macros — triggered by an attribute's name of an element.



4.1. Element macros

Macros that are invoked in-place of an HTML element are called element macros, and their tags are written in uppercase. Parameters for their functionality are passed through attributes, which are usually also in written uppercase.


While they typically can have child elements of any kind, they are considered only by some macros. While some macros copy/evaluate their child elements, others remove them and some only affect the execution environment.


Example: Macro demonstration for language-specific content.

input output
<div>
    <IF lang="de">
        <p>Deutsch</p>
    </IF>
    <ELSE>
        <p>English</p>
    </ELSE>
</div> 		<div>
    <p>English</p>
</div>

Here we decide what content to include in the final document, based on a variable containing the desired language of the document. The decision, which text to include, is done by the flow controll <IF> and <ELSE> macros, which depend on a variable lang.
The variable was not defined within the source document, nor by the command-line options, therefore it is undefined. The resulting content is thus selected from the <ELSE> branch.


List of predefined element macros:

  • <MACRO> — Defines a new macro.
  • <CALL> — Invokes a <MACRO>.
  • <INCLUDE> — Includes content from another file.
  • <SET> — Assigns values to variables.
  • <IF> — Conditionally includes child elements.
  • <ELIF> — Conditionally includes child elements.
  • <ELSE> — Conditionally includes child elements.
  • <FOR> — Includes child elements in a loop.
  • <WHILE> — Includes child elements in a loop.
  • <SET-TAG> — Sets tag of parent element.
  • <GET-TAG> — Sets a variable to parent element's tag.
  • <SET-ATTR> — Sets parent element's attribute value.
  • <GET-ATTR> — Sets variable to value of a parent's attribute.
  • <DEL-ATTR> — Removes attribute from parent element.
  • <SHELL> — Executes shell command.
  • <INFO> — Logs messages to stderr.
  • <WARN> — Logs warning messages to stderr.
  • <ERROR> — Logs error messages to stderr.



<MACRO>...</MACRO>

Defines a new macro.

Macro elements serve as templates for instantiating/evaluating the same HTML element tree multiple times. All <MACRO> elements are referenced globally, i.e. cached and removed from the source document right after the document is parsed and before any instantiation occurs. This means their position in the document doesn't matter.


Attributes:

  • NAME="[name]" represents the name of the new macro, which will be used to invoke it. Any previous macro with the same name is discarded/shadowed by the new one.
  • [variable]=[default_value] represents the name of a variable that is local during the execution of the macro. Variables not listed here or at the calling site (for example in the <CALL> macro), are considered global and any modifications to them remain after the end of the macro execution.
    The expression [default_value] serves as a default value of the new local variable. It is assigned only if it is not defined at the calling site. If [default_value] is not present, the copy of the global variable serves as the default value.
    These attributes also serve as a semantic list of parameters of the macro.


Expression variables within macros are typically global (their modifications carry over after the end of the macro execution), except the list of local variables declared in the attribute list of the element. If the user wishes to use a macro only for computation, they can return the result through a global variable.


A user-defined macro can always be invoked using the <CALL> macro, however if the name (attribute NAME) of the macro begins with an uppercase letter or underscore _ (regex [A-Z_]), it can also be invoked directly by an element's tag, or by an attribute's name of a regular element. If the macro is invoked by an attribute, a local variable VALUE is assigned to the value of the attribute.


When instantiated, all child elements are (recursively) coppied and evaluated:

  • Macro elements are evaluated.
  • Regular HTML elements are coppied.
  • Attributes are coppied and evaluated for any expressions or attribute macros.
  • Text elements are interpolated for expressions.


See also: <CALL>, attribute CALL, <INCLUDE>

Example:

Basic usecase for a custom macro.

input output
<MACRO NAME="func">
    <SET col='if(i == 1, "red", "green")' />
    <li style="color:{col};">Line #{i}</li>
</MACRO>

<ul>
    <FOR i='0' TRUE='i<3' i='i+1' >
        <CALL NAME="func"/>
    </FOR>
</ul> 		<ul>
    <li style="color:green;">Line #0</li>
    <li style="color:red;">Line #1</li>
    <li style="color:green;">Line #2</li>
</ul>

Here we can see how with the help of a <CALL> macro, a list of elements is generated with special cases depending on the global variable i. The variable is global, becausse it is not listed in neither the <MACRO> nor <CALL> macros.

Example:

Behaviours of local and global variables.

input output
<MACRO NAME="func" x y='0'>
    <SET x='x+2' y='y+2' z='z+2'/>
    <p>A: (x,y,z) is ({x}, {y}, {z})</p>
</MACRO>

<SET x='10' y='20' z='30'/>
<CALL NAME="func"/>
<p>B: (x,y,z) is ({x}, {y}, {z})</p>
 <p>A: (x,y,z) is (12, 2, 32)</p>
<p>B: (x,y,z) is (10, 20, 32)</p>

Here we can see how local variables x and y are initialised when calling the macro func and then modified (adding 2). Their modifications are not carried outside of the macro after execution. On the other hand, variable z remains global and its modification remains after the end of the macro.

Example:

Direct macro invocation.

input output
<MACRO NAME="F1">
    <h1>Book named {book}</h1>
</MACRO>
<MACRO NAME="F2">
    <SET-ATTR href="#{VALUE}"/>
    {VALUE}
</MACRO>

<F1 book="Unnamed"/>
<p>
    Go to chapter
    <a F2="main" class="box"/>
</p> 		<h1>Book named Unnamed</h1>
<p>
    Go to chapter
    <a href="#main" class="box">
        main
    </a>
</p>


<CALL/>

Invokes a user-defined macro.

Must be in the form of <CALL NAME=[name] [variable]=[value]/>, where:

  • [name] represents the name of a <MACRO>. It can be an expression.
  • [variable] represents a variable name that is local durring execution of the macro. After the macro finishes, the global (original) variable remains unchanged.
    The set of local variables shadows macro parameters from the called <MACRO>.
  • [value] represents the value of the local variable [variable]. If [value] is not supplied, the variable is cloned from its global counterpart only when the default value of the macro parameter from <MACRO> is also missing.


Other attributes:

  • IF='[expr]' Ignore if expression [expr] evaluates to false.
  • ELIF='[expr]' Ignore if expression [expr] evaluates to false or a conditional on a previous element passed.
  • ELSE Ignore if a conditional on a previous element evaluated passed.


See also: attribute CALL, <MACRO>, <INCLUDE>

Example:

Copying global variable into local variable.

input output
<MACRO NAME="f">
    <SET x='x+2'/>
    <p>x1 is {x}</p>
</MACRO>

<SET x='10'/>
<CALL NAME="f" x/>
<p>x2 is {x}</p>
 <p>x1 is 12</p>
<p>x2 is 10</p>

Variable x is initialized to 10 and in the <CALL> macro it is coppied into its local counterpart. Local x is then modified (12) and inserted into the first <p> element. After the macro concludes, global x remains unchanged (10).

Example:

Creating local variable automatically, with a default value.

input output
<MACRO NAME="f" y='200'>
    <SET y='y+2'/>
    <p>y1 is {y}</p>
</MACRO>

<SET y='20'/>
<CALL NAME="f"/>
<p>y2 is {y}</p>
 <p>y1 is 202</p>
<p>y2 is 20</p>

Variable y is initialized to 20 and since it is not mentioned in the <CALL> macro, the <MACRO> macro initializes its own local variable y=200. Local y is then modified (202) and inserted into the first <p> element. After the macro concludes, global y remains unchanged (20).

Example:

Creating local variable with a value.

input output
<MACRO NAME="f">
    <SET z='z+2'/>
    <p>z1 is {z}</p>
</MACRO>

<SET z='30'/>
<CALL NAME="f" z='300'/>
<p>z2 is {z}</p>
 <p>z1 is 302</p>
<p>z2 is 30</p>

Variable z is initialized to 30 however, the <CALL> macro initializes a new local variable z=300. It is then modified (302) and inserted into the first <p> element. After the macro concludes, global z remains unchanged (30).



<INCLUDE/>

Includes contents of another file.

Files can be included in place of this element macro, thus allowing splitting large files into smaller more managble ones. Each included file is parsed, cached and coupled with its canonical path. The final result of inclusion depends on the type of the file. Content of some files is wrapped with their respective HTML element, thus resulting in valid HTML.


Note:

With .html files, infinitely recursive includes are possible.


File types:

  • Files with .html extensions are treated as file macros. This means that, after parsing, all <MACRO> elements are extracted from the HTML element tree and added to the list of defined user macros, shadowing any existing ones with the same names. The root macro is then executed and all generated HTML elements are appended to the final document.
  • Files with .css extensions are coppied as plain text and appended to the final document.
    Depending on the NO-WRAP attribute, the appended text is wrapped with <style></style> tags.
  • Files with .js extensions are coppied as plain text and appended to the final document.
    Depending on the NO-WRAP attribute, the appended text is wrapped with <script></script> tags.
  • All other files are treated as plain text and coppied to the final document.


Attributes:

  • SRC=[path] Path of the new file relative to the current file. It can be an expression.
  • NO-WRAP Do not wrap contents of certain file types within their respective HTML element:
    • .css files are not wrapped with <style>...</style>.
    • .js files are not wrapped with <script>...</script>.
  • HEADER Only load and cache the file; do not append any content to the final document.
    It is usefull for loading macros at the beginning of the file and later including the file again without the HEADER attribute to include the non <MACRO> content.
  • [variable]=[value] represents a variable name that is local durring execution of the root macro of the included file. After the file is included, the global (original) variable remains unchanged. If [value] is not supplied, the variable is cloned from its global counterpart.


Other attributes:

  • IF='[expr]' Ignore if expression [expr] evaluates to false.
  • ELIF='[expr]' Ignore if expression [expr] evaluates to false or a conditional on a previous element passed.
  • ELSE Ignore if a conditional on a previous element evaluated passed.


See also: <MACRO>, <CALL>, attribute INCLUDE

Example

header.html
<style>
    #header {
        color: red;
    }
</style>
<div id="header">
    <h1>Yet another {name}</h1>
</div>
footer.html
<div id="footer">
    <p>Contacts: none</p>
    <p>Last edited on: 14/10/2025</p>
</div>
input output
<SET name="cookbook"/>
<body>
    <INCLUDE SRC="header.html"/>
    <p>Amazing content!</p>
    <INCLUDE SRC="footer.html"/>
</body> 		<body>
     <style>
        #header {
            color: red;
        }
    </style>
    <div id="header">
        <h1>Yet another cookbook</h1>
    </div>
     <p>Amazing content!</p>
     <div id="footer">
        <p>Contacts: none</p>
        <p>Last edited on: 14/10/2025</p>
    </div>
 </body>


<SET/>

Assigns values to variables.

Must be in the form of <SET [name]=[value] />, where each attribute name and value corresponds to a variable.
The type of the variable is same as the return type of the expression in the attribute value. If the attribute value is within double quotes, it is considered to be a string.


Other attributes:

  • IF='[expr]' Ignore attributes after this attribute, if expression [expr] evaluates to false.
  • ELIF='[expr]' Ignore attributes after this attribute, if expression [expr] evaluates to false or a conditional on a previous element passed.
  • ELSE Ignore attributes after this attribute, if a conditional on a previous element passed.


See also: expressions

Example

input output
<SET x='4' y='6.5' s="apples"/>
<p>I have {int(x*y)} bushels of {s}</p> 		<p>I have 26 bushels of apples</p>


<IF>...</IF>

Conditionally includes child elements.

Must be in the form of <IF [cond]>, where [cond] can either be a boolean check TRUE=[expr] or FALSE=[expr], or a direct variable comparison [var]=[value]. Direct variable comparison is equivalent to TRUE='[var]==[value]'.
Child elements are included in the final document only if all conditionals [cond] are evaluated to true.


See also: <ELIF>, <ELSE>, <SET>, attribute IF, expressions

Example

input output
<IF TRUE='6>3'>
    <p>6 is greater than 3</p>
</IF>
 <p>6 is greater than 3</p>
<SET lang="en"/>
<IF lang="en">
    <p>English</p>
</IF>
 <p>English</p>


<ELIF>...</ELIF>

Includes child elements if previous condition macros fail

Must follow a previous <IF> macro and be in the form of <ELIF [cond]>, where [cond] can either be a boolean check TRUE=[expr] or FALSE=[expr], or direct variable comparison [var]=[value]. Direct variable comparison is equivalent to TRUE='[var]==[value]'.
Child elements are included in the final document only if a previous <IF> macro failed and all conditionals [cond] of this macro are evaluated to true.


See also: <IF>, <ELSE>, <SET>, attribute ELIF, expressions

Example

input output
<SET lang="en"/>

<IF lang="de">
    <p>Deutsch</p>
</IF>
<ELIF lang="en">
    <p>English</p>
</ELIF>
 <p>English</p>


<ELSE>...</ELSE>

Includes child elements if previous condition macros fail.

Must follow a previous <IF> or <ELIF> macro.
Child elements are included in the final document only if a previous <IF> or <ELIF> macro failed.


See also: <IF>, <ELIF>, <SET>, attribute ELSE, expressions

Example

input output
<SET lang="en"/>

<IF lang="de">
    <p>Deutsch</p>
</IF>
<ELSE>
    <p>English</p>
</ELSE>
 <p>English</p>


<FOR>...</FOR>

Evaluates child elements for a set amount of times.

Must be in the form of <FOR [init] [cond] [inc]>, where:

  • [init] represents setting/initializing a variable in the form [name]=[value], where the attribute name is the name of the variable, and the attribute value is the value of the variable.
  • [cond] represents a conditional attribute TRUE='[expr]' or FALSE='[expr]'. The loop ends when the expression of the attribute value evaluates to false, or true respectively.
  • [inc] represents setting a variable to a new value, in the form of [name]=[expr], after each iteration of the loop.


Other attributes:

  • IF='[expr]' Ignore if expression [expr] evaluates to false.
  • ELIF='[expr]' Ignore if expression [expr] evaluates to false or a conditional on a previous element passed.
  • ELSE Ignore if a conditional on a previous element evaluated passed.


See also: <WHILE>, expressions

Example

input output
<FOR i='0' TRUE='i < 3' i='i+1'>
    <p>{i}</p>
</FOR> 		<p>0</p>
<p>1</p>
<p>2</p>


<WHILE>...</WHILE>

Repeatedly evaluates child elements.

Must be in the form of <WHILE [cond]>, where:

  • [cond] represents a conditional attribute TRUE='[expr]' or FALSE='[expr]'. The loop ends when the expression of the attribute value evaluates to false, or true respectively.


Other attributes:

  • IF='[expr]' Ignore if expression [expr] evaluates to false.
  • ELIF='[expr]' Ignore if expression [expr] evaluates to false or a conditional on a previous element passed.
  • ELSE Ignore if a conditional on a previous element evaluated passed.


See also: <FOR>, expressions

Example

input output
<SET i='0'/>
<WHILE TRUE='i < 3'>
    <p>{i}</p>
    <SET i='i+1'/>
</WHILE> 		<p>0</p>
<p>1</p>
<p>2</p>

<GET-ATTR/>

Set variable to value of a parent's attribute.

Must be in the form of <GET-ATTR [variable]=[attribute] /> where:

  • [variable] represents the name of the variable which will hold the value of the attribute.
  • [attribute] represents the name of the parent element's attribute from which to retrieve the value.


The type of the variable [variable] is string, as attribute values are strings. If the attribute is not found however, the variable is set to an integer 0.
If there are multiple attributes with the same name, the value of the first one is used.


Other attributes:

  • IF='[expr]' Ignore attributes after this attribute, if expression [expr] evaluates to false.
  • ELIF='[expr]' Ignore attributes after this attribute, if expression [expr] evaluates to false or a conditional on a previous element passed.
  • ELSE Ignore attributes after this attribute, if a conditional on a previous element passed.


See also: <SET-ATTR>, <DEL-ATTR>

Example

input output
<div id="box">
    <GET-ATTR x="id"/>
    <SET-ATTR class='x'/>
</div> 		<div id="box" class="box">
</div>

Must be in the form of <DEL-ATTR [name] /> where:

Other attributes:

</div>


<SHELL/>

Execute shell command.

Must be in the form of <SHELL>[cmd]</SHELL> where:

  • [cmd] represents the command that will be passed to a shell. The command is not interpolated for any expressions.


The underlying shell for executing the command is bash.


Attributes:

  • VARS="[name,...]" Set shell environment variables to the comma-separated-list of variable names. This is the only way to share variables, as the command [cmd] is not interpolated. Listed variables are accessed by the $[name] operator.
  • STDOUT="[VOID|TEXT|[variable]]" Determines what to do with the stdout stream (default TEXT):
    • VOID Discards all output.
    • TEXT Insert shell output as text into the final document. The last newline of the output is trimmed.
    • [variable] Store shell output into a variable named [variable]. The last newline of the output is trimmed.


Other attributes:

  • IF='[expr]' Ignore if expression [expr] evaluates to false.
  • ELIF='[expr]' Ignore if expression [expr] evaluates to false or a conditional on a previous element passed.
  • ELSE Ignore if a conditional on a previous element evaluated passed.


See also: <SET>

Example

input output
<SET t='0' fmt="%d/%m/%Y" />
<p>
    The world was created on:
    <SHELL VARS="t,fmt">date -d "@$t" +"$fmt"</SHELL>
</p> 		<p>

4.2. Attribute macros

Macros that are invoked in-place of an element's attribute are called attribute macros, and their names are written in uppercase. They typically serve as short forms of element macros.


Example: Macro attribute demonstration for language-specific text.

input output
<div>
    <p IF='lang=="de"'>Deutsch</p>
    <p ELSE>English</p>
</div> 		<div>
    <p>English</p>
</div>

Here we conditionally include the first p element, but only if the expression in the IF macro attribute evaluates to true. Otherwise, we include the second p element.


List of predefined attribute macros:

  • IF — Conditionally include element.
  • ELIF — Conditionally inlude element.
  • ELSE — Conditionally inlude element.
  • CALL — Invoke a user-defined macro inside the current element.
  • INCLUDE — Include contents of another file as child elements of the current element.



IF=...

Conditionally include element.

Must be in the form of <... IF='[expr]' ...>, where:

  • [expr] represents the expression that is evaluated to a boolean.


If the expression expr evaluates to true, the element is included in the final document; otherwise it is discarded. This attribute macro can also be found on almost all macro elements, where it performs the same function with a small caveat:

  • Evaluation of an element macro is terminated only after this attribute macro is encountered and evaluated to false; any previous attributes can already have an effect on the final document. If such behaviour is unwanted (for example in a <SET/> macro), the IF attribute should be written before all others.


See also: attribute ELIF, attribute ELSE, expressions

Example

input output
<SET n='1'/>
<p IF='n==1'>one</p>
<p ELIF='n==2'>two</p>
<p ELSE>none</p> 		<p>one</p>


ELIF=...

Include element if previous condition atribute macro fails.

Must be in the form of <... ELIF='[expr]' ...>, where:

  • [expr] represents the expression that is evaluated to a boolean.


The element is included in the final document only if an attribute macro IF or ELIF on a previous element evaluates to false, and the expression expr evaluates to true. This attribute macro can also be found on almost all macro elements, where it performs the same function with a small caveat:

  • Evaluation of an element macro is terminated only after this attribute macro is encountered and is determined, that the element should be discarded. Any previous attributes can already have an effect on the final document. If such behaviour is unwanted (for example in a <SET/> macro), the ELIF attribute should be written before all others.


See also: attribute IF, attribute ELSE, expressions

Example

input output
<SET n='2'/>
<p IF='n==1'>one</p>
<p ELIF='n==2'>two</p>
<p ELSE>none</p> 		<p>two</p>


ELSE

Include element if previous condition atribute macro fails.

Must be in the form of <... ELSE ...>, where:

  • [expr] represents the expression that is evaluated to a boolean.


The element is included in the final document only if an attribute macro IF or ELIF on a previous element evaluates to false. This attribute macro can also be found on almost all macro elements, where it performs the same function with a small caveat:

  • Evaluation of an element macro is terminated only after this attribute macro is encountered and is determined, that the element should be discarded. Any previous attributes can already have an effect on the final document. If such behaviour is unwanted (for example in a <SET/> macro), the ELSE attribute should be written before all others.


See also: attribute IF, attribute ELSE

Example

input output
<SET n='3'/>
<p IF='n==1'>one</p>
<p ELIF='n==2'>two</p>
<p ELSE>none</p> 		<p>none</p>


CALL=...

Invoke a user-defined macro inside the current element.

Must be in the form of <... CALL=[name] ...>, where:

  • [name] represents the name of a <MACRO> that will be invoked. It can be an expression.


The invocation occurs immediately after encountering the attribute. This means that some attributes might not have fully coppied to the final document. This would affect macros such as <DEL-ATTR>, as they would not be able to remove attributes that do not exist yet. If such behaviour is unwanted, the attribute macro should be written after all others.


After invoking the macro, all child elements are evaluated/appended to the final document as per the standard procedure. This attribute macro is esentially an attribute version of the <CALL> macro. A big shortcoming of this attribute macro is the inability to pass any arguments to the macro.


See also: <CALL>, <MACRO>



INCLUDE=[path]

Include contents of another file as child elements of the current element.

Works in the same way as the element macro <INCLUDE>, with the exception that included content is never wrapped (NO-WRAP) and the user cannot declare any local variables.


The file is included immediately after encountering the attribute. This means that some attributes might not have fully coppied to the final document. This would affect macros such as <DEL-ATTR>, as they would not be able to remove attributes that do not exist yet. If such behaviour is unwanted, this macro should be written after all others.


See also: <INCLUDE>

Example

content.html
<IF lang="de">
    <p IF='i==1'>Ein</p>
    <p ELIF='i==2'>Zwei</p>
    <p ELIF='i==3'>Drei</p>
</IF>
<ELSE>
    <p IF='i==1'>One</p>
    <p ELIF='i==2'>Two</p>
    <p ELIF='i==3'>Three</p>
</ELSE>
input output
<SET lang="en"/>
<body lang="{lang}">
    <ol>
        <FOR i='1' TRUE='i<=3' i='i+1'>
            <li INCLUDE="content.html"/>
        </FOR>
    </ol>
</body> 		<body lang="en">
    <ol>
        <li>
             <p>One</p>
         </li>
        <li>
             <p>Two</p>
         </li>
        <li>
             <p>Three</p>
         </li>
    </ol>
</body>



5. Expressions

In certain contexts, expressions can be used to interpolate HTML text nodes, or to provide computed values to macros. They consist of constants, expression operators, variables and functions.


There are only 3 fundamental data types that can be evaluated from expressions:

  • integer: long, 64 bit,
  • floating point number: double, 64 bit,
  • string: char[].


For purposes of easier explanation and documentation, 2 implicit data types are defined:

  • number: int or float,
  • boolean: int, where logical false equals 0 and true equals 1.



5.1. Expression context

Expressions can be defined and evaluated within HTML attribute values or HTML text nodes. HTML attribute values, quoted with single quotes (''), are directly evaluated as expressions and then converted to text before they are appended to the final document. Attribute values, quoted with double quotes (""), are evaluated as strings/plain text.


Any plain text or string attribute values (quoted with double quotes "...") are interpolated for any expressions contained within curly brackets {}. Content within the curly brackets is evaluated as an expression and it's result is stringified and inserted in-place of the expression in the final document.


In order to escape an expression (to consider it as just text and not an expression), it must be prefixed with a backslash: \{...}.


Example: Attribute expressions and text interpolation.

input output
<p attr="text {1 + 2}"></p>
<p attr='2 * 2'></p>
<p>There are {9.0 / 2} apples.</p>
<p>Escaped expression \{1+1}</p> 		<p attr="text 3"></p>
<p attr="4"></p>
<p>There are 4.5 apples.</p>
<p>Escaped expression {1+1}</p>


5.2. Constants

Regex descriptions for types of constants:

  • int: -?[0-9]+
  • float: -?[0-9]+[.][0-9]*
  • string: ".*?"|'.*?' (All text between either single ' or double " quotes)


Some predefined variables are available for certain numeric/logical constants:

Name Value
false 0
true 1
pi π or 3.14159...


5.3. Operators

Expression operators consist of standard unary and binary operators:

  • Unary operatos: - (arithmetic negation), ! (boolean negation)
  • Arithmetic binary operators: +, -, *, /, %
  • Logical operators: && (and), || (or)
  • Comparison binary operators: ==, !=, <, <=, >, >=


For some operations with operands of different types, some of the operands might get coerced/casted into other types. For example, adding a number to a string results in the number being stringified and appended to the string operand. Another case is coercing a string into a boolean where it evaluates to true when it's not empty.


Here is a list of notable cases of operations between different types:

Operation Description Example
string + any The second operand is converted into a string and then appended to the main string operand. "x is " + 3 → "x is 3"
string * number Produces a string that is appended to itself n times, where the number is casted into an int. "a" * 3 → "aaa"
string - number Shortens the string by n characters. "abc" - 2 → "a"
!string,
string || any,
string && any 		The string is converted into a bool, equivalent to len(string) != 0 "a" || "" → true || false → 1
!number,
number || any,
number && any 		The number is converted into a bool, equivalent to number != 0 !10 → !true → 0
float % number Produces a remainder the same way it is done with an int. 3.6 % 1.25 → 1.1


For invalid operations between certain types, either 0 or a copy of the first operand is produced.



5.4. Variables

Variable identifiers comprise of alpha-numerical and underscore characters, where the first character cannot be a digit. The equivalent regex would be: [a-zA-Z_][a-zA-Z0-9_]*. They are defined/assigned with the <SET/> macro and can be referenced globally from all expressions. The types of their values are dynamic and correspond to their most recently assigned value type.


At program start, some variables get predefined values, as explained in the chapter on constants. The user can also provide more predefined variables through command-line options in the form of name=value, as explained in the command-line interface chapter.


Example: Defining and using variables.

input output
<SET s="header"/>
<p class='s'>My title</p>

<SET x='2' y='3'/>
<p>My text number {x * (y + 1)}</p> 		<p class="header">My title</p>
<p>My text number 8</p>
