{ci}. Macros

Certain HTML tags are not coppied to the final document. Instead they perform other functions that modify the document. Such tags are called (HTML) macros and are all written in uppercase. Parameters for their functionality are passed by HTML attributes, which are usually also in uppercase.


Example: Macro demonstration for language-specific text.

input output
{l}div{r}
{t}{l}IF lang="de"{r}
{t*2}{l}p{r}Deutsch{l}/p{r}
{t}{l}/IF{r}
{t}{l}ELSE{r}
{t*2}{l}p{r}English{l}/p{r}
{t}{l}/ELSE{r}
{l}/div{r} 		{l}div{r}
{t}{l}p{r}English{l}/p{r}
{l}/div{r}

Here we decide language-specific text for the final document. The decicion, which text to include, is done by the flow controll IF and ELSE macros, which depend on a variable lang.


Some macro attributes can also be applied to regular HTML tags. These attributes are also not included in the final document.


Example: Macro attribute demonstration for language-specific text.

input output
{l}div{r}
{t*2}{l}p IF='lang=="de"'{r}Deutsch{l}/p{r}
{t*2}{l}p ELSE{r}English{l}/p{r}
{l}/div{r} 		{l}div{r}
{t}{l}p{r}English{l}/p{r}
{l}/div{r}

Here we conditionally include the first p element, but only if the expression in the IF macro attribute evaluates to true. Otherwise, we include the second p element.



{ci}.{cii}. List of macros

  • {l}MACRO{r} — Defines a new macro.
  • {l}CALL{r} — Invokes a {l}MACRO{r}.
  • {l}INCLUDE{r}
  • {l}SET{r} — Assigns values to variables.
  • {l}IF{r} — Conditionally includes child elements.
  • {l}ELIF{r} — Extends {l}IF{r}.
  • {l}ELSE{r} — Extends {l}IF{r}.
  • {l}FOR{r} — Include child elements in a loop.
  • {l}WHILE{r} — Include child elements in a loop.
  • {l}SET-TAG{r}
  • {l}GET-TAG{r}
  • {l}SET-ATTR{r}
  • {l}GET-ATTR{r}
  • {l}DEL-ATTR{r}
  • {l}SHELL{r}
  • {l}INFO{r} — Log messages to stderr.
  • {l}WARN{r} — Log warning messages to stderr.
  • {l}ERROR{r} — Log error messages to stderr.

{desc}

Must be in the form of {l}MACR NAME="[name]"{r}, where:

  • [name] represents the name of the new macro, which will be used to invoke it. Only the latest macro with the same name is cached.


Macro elements serve as templates for instantiating a large number of simmilar HTML element trees. All {l}MACRO{r} elements are referenced globally, i.e. cached and removed from the source document right after the document is parsed and before any instantiation occurs. This means their position in the document doesn't matter.


All files have an anonymous {l}MACRO{r} root element.


When instantiated, all child elements are (recursively) coppied and evaluated:

  • Macro elements are evaluated.
  • Regular HTML elements are coppied.
  • Attributes are coppied and evaluated for any expressions or attribute macros.
  • Text elements are interpolated for expressions.


See also: {l}CALL{r}, {l}INCLUDE{r}

Example

input output
{l}MACRO NAME="func"{r}
{t}{l}SET col='if(i == 1, "red", "green")' /{r}
{t}{l}li style="color:\{col};"{r}Line #\{i}{l}/li{r}
{l}/MACRO{r}

{l}ul{r}
{t}{l}FOR i='0' TRUE='i{l}3' i='i+1' {r}
{t*2}{l}CALL NAME="func"/{r}
{t}{l}/FOR{r}
{l}/ul{r} 		{l}ul{r}
{t}{l}li style="color:green;"{r}Line #0{l}/li{r}
{t}{l}li style="color:red;"{r}Line #1{l}/li{r}
{t}{l}li style="color:green;"{r}Line #2{l}/li{r}
{l}/ul{r}

Must be in the form of {l}CALL NAME="[name]"{r}, where:

  • [name] represents the name of a {l}MACRO{r}.

See also: {l}MACRO{r}, {l}INCLUDE{r}

Must be in the form of {l}INCLUDE SRC="[path]"/{r}, where:

  • [path] represents the path of the new file relative to the current file.


Parsing of the included file depends on the file type, which is determined from the file extension. If the file type is ambiguous, it can be forced with a specialised {l}INCLUDE-[type]{r} macros:

  • {l}INCLUDE-HTML/{r} for .html files.
    The file is parsed as an HTML macro, i.e. the root element is invoked as a macro after parsing.
    All {l}MACRO{r} elements are added to the global list of defined macros (overwriting existing ones).
  • {l}INCLUDE-CSS/{r} for style .css files.
    The file is processed for any CSS expressions.
    The attribute version of this macro is more usefull, as it allowes wrapping the content in {l}style{r}{l}/style{r} tags.
    May be subjected to optimizations in future versions.
  • {l}INCLUDE-JS/{r} for JavaScript .js files.
    Mainly untouched; handled as plain text.
    The attribute version of this macro is more usefull, as it allowes wrapping the content in {l}script{r}{l}/script{r} tags.
    May be subjected to optimizations in future versions.
  • {l}INCLUDE-TXT/{r} for plain text files.
    Untouched and coppied to the final document.


Every included file is cached which means that parsing is done only once per file.
The same file can be included multiple times which means that the user must prevent infinitely recursive includes.


See also: {l}CALL{r}

Must be in the form of {l}SET [name='value'] /{r}, where each attribute name and value corresponds to a variable.
The type of the variable is same as the return type of the expression in the attribute value. If the attribute value is within double quotes, it is considered to be a string.
All variables are global.

Example

input output

{l}p{r}I have \{int(x*y)} bushels of \{s}{l}/p{r} 		{l}p{r}I have 26 bushels of apples{l}/p{r}

Must be in the form of {l}IF [cond]{r}, where [cond] can either be a boolean check TRUE='...' or FALSE='...', or direct variable comparison var='value'. Direct variable comparison is equivalent to TRUE='var==value'.
Child elements are included in the final document only if all conditionals [cond] are evaluated to true.


See also: {l}ELIF{r}, {l}ELSE{r}, {l}SET{r}, expressions

Example

input output
{l}IF TRUE='6>3'{r}
{t}{l}p{r}6 is greater than 3{l}/p{r}
{l}/IF{r}
 {l}p{r}6 is greater than 3{l}/p{r}
{l}SET lang="en"/{r}
{l}IF lang="en"{r}
{t}{l}p{r}English{l}/p{r}
{l}/IF{r}
 {l}p{r}English{l}/p{r}

Must follow a previous {l}IF{r} macro and be in the form of {l}ELIF [cond]{r}, where [cond] can either be a boolean check TRUE='...' or FALSE='...', or direct variable comparison var='value'. Direct variable comparison is equivalent to TRUE='var==value'.
Child elements are included in the final document only if a previous {l}IF{r} macro failed and all conditionals [cond] of this macro are evaluated to true.


See also: {l}IF{r}, {l}ELSE{r}, {l}SET{r}, expressions

input output

{t}{l}p{r}Deutsch{l}/p{r}
{l}/IF{r}
{l}ELIF lang="en"{r}
{t}{l}p{r}English{l}/p{r}
{l}/ELIF{r}
 {l}p{r}English{l}/p{r}

Must follow a previous {l}IF{r} or {l}ELIF{r} macro.
Child elements are included in the final document only if a previous {l}IF{r} or {l}ELIF{r} macro failed.

Must be in the form of {l}FOR [init] [cond] [inc]{r}, where:

  • [init] represents setting a variable in the form name='value', where the attribute name is the name of the variable, and the attribute value is the value of the variable.
  • [cond] represents a conditional attribute TRUE='...' or FALSE='...'. The loop ends when the expression of the attribute value evaluates to false, or true respectively.
  • [inc] represents setting a variable to a new value, in the form of name='value', after each iteration of the loop.

Must be in the form of {l}SET-ATTR [name]=[value] /{r}, where:

  • [name] represents the name of the attribute that will be added or overwritten on the parent element.
  • [value] represents the value or evaluatable expression of the new attribute.


Other attributes:


See also: {l}GET-ATTR{r}, {l}DEL-ATTR{r}

Example

input output

Must be in the form of {l}GET-ATTR [variable]=[attribute] /{r} where:

  • [variable] represents the name of the variable which will hold the value of the attribute.
  • [attribute] represents the name of the parent element's attribute from which to retrieve the value.


The type of the variable [variable] is string, as attribute values are strings. If the attribute is not found however, the variable is set to an integer 0.
If there are multiple attributes with the same name, the value of the first one is used.

IF='[expr]' Ignore if expression [expr] evaluates to false. IF='[expr]' Ignore attributes after this attribute, if expression [expr] evaluates to false. ELIF='[expr]' Ignore if expression [expr] evaluates to false or a conditional on a previous element passed.
