{ci}. Expressions

In certain contexts, expressions can be used to interpolate HTML text nodes, or to provide computed values to macros. They consist of constants, operators, variables and functions.


There are 5 fundamental data types that can be evaluated from expressions:

  • null: nothing,
  • int: long, 64 bit,
  • float: double, 64 bit,
  • string: char[], maximum length of 232.
  • object: array and map.


For purposes of easier explanation and documentation, 4 implicit data types are defined:

  • number: int or float,
  • bool: int, where logical false equals 0 and true equals 1,
  • array: object but referencing only the array part,
  • map: object but referencing only the map part.


To further clarify the object data type, it contains both an array and a map (string → any mapping, also known as a dictionary). When using the indexer operator [...], the underlying value retrieval depends on the type of the index:

  • If the index is a number, it is retrieving values from the array.
  • If the index is a string, it is retrieving values from the map.



{ci}.{cii}. Expression context

Expressions can be defined and evaluated within HTML attribute values or HTML text nodes. HTML attribute values, quoted with single quotes (''), are directly evaluated as expressions and then converted to text before they are appended to the final document. Attribute values, quoted with double quotes (""), are evaluated as strings/plain text.


Any plain text or string attribute values (quoted with double quotes "...") are interpolated for any expressions contained within curly brackets \{}. Content within the curly brackets is evaluated as an expression and then converted to text and inserted in-place of the expression in the final document.


In order to escape an expression (to consider it as just text and not an expression), it must be prefixed with a backslash: \\\{...}.


Example: Attribute expressions and text interpolation.

input output
{l}p attr="text \{1 + 2}"{r}{l}/p{r}
{l}p attr='2 * 2'{r}{l}/p{r}
{l}p{r}There are \{9.0 / 2} apples.{l}/p{r}
{l}p{r}Escaped expression \\\{1+1}{l}/p{r} 		{l}p attr="text 3"{r}{l}/p{r}
{l}p attr="4"{r}{l}/p{r}
{l}p{r}There are 4.5 apples.{l}/p{r}
{l}p{r}Escaped expression \{1+1}{l}/p{r}


{ci}.{cii}. Values

Descriptions for types of constants or value constructors:

  • int: -?[0-9]+
  • float: -?[0-9]+[.][0-9]*
  • string: ".*?"|'.*?' All text between either single ' or double " quotes
  • object: [x, y, ..., "name_1": z, "name_2": w, ...] List of comma-separated values between two square brackets []. If a value is preceeded by a string and a colon :, it defines an entry in the map, otherwise it is the n-th element of the array.


Example: Constructing an object value.

input output
{l}SET a='[0,10,20]'{r}{l}/SET{r}
{l}SET m='["x": 30, "y": 40]'{r}{l}/SET{r}
{l}SET am='[0, 10, 20, "x": 30, "y": 40]'{r}{l}/SET{r}
{l}p{r}\{a[1]}{l}/p{r}
{l}p{r}\{m["y"]}{l}/p{r}
{l}p{r}\{am[2] + am["x"]}{l}/p{r}
 {l}p{r}10{l}/p{r}
{l}p{r}40{l}/p{r}
{l}p{r}50{l}/p{r}

{ci}.{cii}. Operators

Expression operators consist of unary and binary operators:

  • Unary operatos: - (arithmetic negation), ! (boolean negation)
  • Arithmetic or overloaded binary operators: +, -, *, /, %
  • Logical binary operators: && (and), || (or)
  • Comparison binary operators: ==, !=, <, <=, >, >=
  • Indexer: []


For some operations with operands of different types, some of the operands might get coerced/casted into a common type. For example, adding a number to a string results in the number being stringified and appended to the string operand. Another case is coercing a string into a boolean with logical operators where it evaluates to true when it's not empty. For invalid operations between certain types, either the first operand or null is returned.

{ci}.{cii}. Variables

Variable identifiers comprise of alpha-numerical and underscore characters, where the first character cannot be a digit. The equivalent regex would be: [a-zA-Z_][a-zA-Z0-9_]*. They are defined/assigned with the <SET/> macro and can be referenced globally from all expressions. The types of their values are dynamic and correspond to their most recently assigned value type.


At program start, some variables get predefined values, as explained in the chapter on values. The user can also provide more predefined variables through command-line options in the form of name=value, as explained in the command-line interface chapter.


Example: Defining and using variables.

input output
<SET s="header"/>
<p class='s'>My title</p>

<SET x='2' y='3'/>
<p>My text number \{x * (y + 1)}</p> 		<p class="header">My title</p>
<p>My text number 8</p>
