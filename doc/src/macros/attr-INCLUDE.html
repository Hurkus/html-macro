INCLUDE=...

Include contents of another file as child elements of the current element.

Must be in the form of {l}... INCLUDE="[name]" ...{r} or {l}... INCLUDE-[type]="[name]" ...{r}, where:

  • [path] represents the path of the new file relative to the current file.
  • [type] represents the forced interpretation of the file being included. Optional only when the type of the file can be determined from the path extension.
    Valid types are:
    • HTML for .html files.
      The file is parsed as an HTML macro, i.e. the root element is invoked as a macro after parsing.
      All {l}MACRO{r} elements are added to the global list of defined macros (overwriting existing ones).
    • CSS for style .css files.
      The file is processed for any CSS expressions.
      May be subjected to optimizations in future versions.
    • JS for JavaScript .js files.
      Mainly untouched; handled as plain text.
      May be subjected to optimizations in future versions.
    • TXT for plain text files.
      Untouched; coppied to the final document.


The file is included immediately after encountering the attribute. This means that some attributes might not have fully coppied to the final document. This would affect macros such as {l}DEL-ATTR{r}, as they would not be able to remove attributes that do not exist yet. If such behaviour is unwanted, the attribute macro should be written after all others.

After including the file, all child elements are evaluated/appended to the final document as per the standard procedure. This attribute macro is esentially an attribute version of the {l}INCLUDE{r} macro.


See also: {l}INCLUDE{r}
