{l}INCLUDE/{r}

Includes contents of another file.

Files can be included in place of this element macro, thus allowing splitting large files into smaller more managble ones. Each included file is parsed, cached and coupled with its canonical path. The final result of inclusion depends on the type of the file. Content of some files is wrapped with their respective HTML element, thus resulting in valid HTML.


Note:

With .html files, infinitely recursive includes are possible.


File types:

  • Files with .html extensions are treated as file macros. This means that, after parsing, all {l}MACRO{r} elements are extracted from the HTML element tree and added to the list of defined user macros, shadowing any existing ones with the same names. The root macro is then executed and all generated HTML elements are appended to the final document.
  • Files with .css extensions are coppied as plain text and appended to the final document.
    Depending on the NO-WRAP attribute, the appended text is wrapped with {l}style{r}{l}/style{r} tags.
  • Files with .js extensions are coppied as plain text and appended to the final document.
    Depending on the NO-WRAP attribute, the appended text is wrapped with {l}script{r}{l}/script{r} tags.
  • All other files are treated as plain text and appended to the final document.


Attributes:

  • SRC=[path] Path of the new file relative to the current file. It can be an expression.
  • NO-WRAP Do not wrap contents of certain file types within their respective HTML element:
    • .css files are not wrapped with {l}style{r}{l}/style{r}.
    • .js files are not wrapped with {l}script{r}{l}/script{r}.
    • Others aren't even wrapped.
  • [variable]=[value] represents a variable name that is local durring execution of the root macro of the included file. After the file is included, the global (original) variable remains unchanged. If [value] is not supplied, the variable is cloned from its global counterpart.


Other attributes:


See also: {l}MACRO{r}, {l}CALL{r}, attribute INCLUDE

Example

header.html
{l}style{r}
{t}#header {
{t*2}color: red;
{t}}
{l}/style{r}
{l}div id="header"{r}
{t}{l}h1{r}Yet another \{name}{l}/h1{r}
{l}/div{r}
footer.html
{l}div id="footer"{r}
{t}{l}p{r}Contacts: none{l}/p{r}
{t}{l}p{r}Last edited on: 14/10/2025{l}/p{r}
{l}/div{r}
input output

{l}body{r}
