{ci}. Command-line interface

The program requires a path to one input file, which will be parsed, processed and then piped via stdout.


General usage would be: html-macro [options] [variables] {l}path{r}, where:

  • [options] represents the optional list of program options.
  • [variables] represents the list of predefined variables used in expressions. They are written in the form of [name]=[value].
  • {l}path{r} represents the mandatory path to the input file (typically a .html file). The path cannot contain the = character, as that would be considered a variable declaration.


List of program [options]:

Long option Short option Description
--help -h Print program information and a short manual/list of all options.
--include {l}path{r} -i {l}path{r} Add path to the list of paths that are searched when including files using a relative path with the {l}INCLUDE{r} element macro. The path must lead to an existing directory.
--output {l}path{r} -o {l}path{r} Write output directly to a file instead of stdout. Errors, warnings and general information are still piped to stderr.
--type {l}type{r} -t {l}type{r} Force input file to be treated as a different file type, where {l}type{r} can be html, css, js or txt.
--compress {l}type{r} -c {l}type{r} Compress output by removing unecessary spaces and other constructs. The {l}type{r} can be none, html, css or all.
-x Discard any output except errors and warnings. This option can be used to check for errors without generating any output.
--dependencies -d Scan the document and extract all paths from various macros that reference other files (such as {l}INCLUDE{r}) and print them. This is usefull when generating dependency files with make.
