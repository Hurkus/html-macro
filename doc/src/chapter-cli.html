{ci}. Command-line interface

The program requires a path to one input file, which will be parsed, processed and then piped via stdout.


General usage would be: html-macro [options] [variables] {l}path{r}, where:

  • [options] represents the optional list of program options described below.
  • [variables] represents the list of predefined variables used in expressions. They are written in the form of [name]=[value].
  • {l}path{r} represents the mandatory path to the input file (typically a .html file). The path cannot contain an = character, as that would be considered a variable assignment.


List of program [options]:
Long option Short option Description
--help -h Print program information and a short manual/list of all options.
--include {l}path{r} -i {l}path{r} Add path to the list of paths that are searched when including files using a relative path with the {l}INCLUDE{r} element macro. The path must lead to an existing directory.
--output {l}path{r} -o {l}path{r} Write output directly to a file instead of stdout. Errors, warnings and general information are still piped to stderr.
-x Discard the stdout stream and/or do not write any output to a file. This option can be used to check for errors without generating any output.
